# コミットメッセージ規約

pika のコミットメッセージは **日本語**で書く（CLAUDE.md の方針）。

- 全体で **3 行以内**に収める
- **1 行目**: `type: 要約` の形。`type` は半角英小文字、要約は日本語で簡潔に
  - 例: `feat: 差分ビューに確認済みトグルを追加`
  - 例: `fix: watcher の自己保存イベントを抑制し再描画ループを解消`
  - `type` の語彙: `feat`(機能) / `fix`(修正) / `docs`(文書) / `refactor` / `test` / `perf`(性能) / `build`(ビルド・依存) / `ci` / `chore`(雑務)
- **2〜3 行目（任意）**: **なぜ**変更したかを書く。what は diff で分かるので、理由・背景だけに絞る
- 要件・設計に関わる変更は、対応する `docs/requirements.md` / `docs/design.md` の章を本文で参照すると追跡しやすい
