E. 状態記号の体系（docs/ui-design.md へ正典化する候補）

±差分あり（未読） — ベースラインから変化。強調色で目立たせる（ツリー・タブ共通）
±配下に差分あり — 子孫に差分ありのフォルダ（淡色で伝播）
●未保存 — タブの閉じる位置・強調色（位置＋色で未読と弁別）
◆新規ファイル — ◆（緑の菱形）。差分行の＋とは別記号で混同回避
名削除済み — 取り消し線（淡色）。差分行の−とは別
＋差分追加 — 緑背景＋＋記号＋行内下線（差分は色OK）
−差分削除 — 赤背景＋−記号＋行内下線（差分は色OK）
✓確認済みにする — モノトーンの枠線ボタン
⚠衝突のみ色強調 — 左に細い警告バー。他通知はモノトーン

重畳時の表示優先: 削除済み ＞ 未保存 ＞ 未読（req 5.3）。色を使うのは差分と衝突警告だけ、他は形・位置・濃淡で弁別。ハイコントラスト追従。
