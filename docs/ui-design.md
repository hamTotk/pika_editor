## 1. デザイン原則

実装中の見た目の判断はこの優先順位に従う。

1. **モノトーン基調＋僅かに色相をずらしたグレー**（Obsidian 路線）。装飾を削ぎ、情報密度で勝負する（ツリー・タブ・差分は密度が要る）。階層は罫線＋余白＋面の明度差で作り、影には頼らない（Win11 Fluent／ダーク両対応）。
2. **色を使うのは「差分」と「特に重要な強調」だけ**。
   * 差分（追加=緑／削除=赤）= 差分ビュー・差分行の主役。
   * 強調色（アクセント青）= 編集者の注目を促すもの（未保存マーク・カーソル位置・ファイルの「差分あり」マーク・差分トグルON）。
   * 警告色（amber/alert）= データ損失リスク（衝突通知の左バーのみ）。
   * それ以外（選択行・タブ・モード切替・ステータス・他の通知・ファイルタイプアイコン）はすべてモノトーン。
   * **【2026-06-28 改訂・ユーザー判断】エディタの構文ハイライトはこのモノトーン規則の唯一の例外**とし、`--syntax-*` トークンで md/HTML/コードに色数を解禁する（見出し/キーワード/文字列/型/タグ、および装飾記号 `#`/`>`/`*`/背鈎 等にも配色）。差分（緑/赤）・強調（accent）・警告（alert）の色運用とは**彩度を抑えて視覚的に競合させない**。UIシェル（ツリー/タブ/ステータス/通知/ファイルタイプアイコン）は従来どおりモノトーンを維持する。トークン値は §2 の `--syntax-*` 表。
3. **色だけに依存しない**。状態はすべて形・位置・記号でも弁別できる（ハイコントラスト／モノクロ／色覚多様性に追従。要件11.5）。
4. **ネイティブの見た目に馴染ませる**。ネイティブコントロールの既定描画を尊重し、カスタムは配色テーマ・状態マーク・アイコン・余白に留める。
5. **新規フォントを同梱しない**（軽量原則。配布30MB）。UI はシステムフォント、コードは Windows 同梱の等幅。

AI slop（紫グラデ・glow・カード乱用・色付きストライプ等）を避ける。余白・整列・色の一つ一つに理由を言えることを基準とする。

## 3. タイポグラフィ

* `font-ui`：`"Segoe UI Variable Text", "Segoe UI", system-ui`（システム同梱・非同梱）。UI 全般 13–14px。
* `font-mono`：`"Cascadia Code", "Cascadia Mono", Consolas`（Windows 同梱・非同梱）。エディタ・差分・コード。
* Markdown プレビュー本文：15.5px／行間 1.65／行長は最大 760px（45–75字目安）。見出し h1 26px・h2 20px。
* font-weight 400 未満は使わない。強調は太字＋色で。
* ツリー/タブの行高は 25px（密度重視）。

## 4. 余白・角丸・密度

* スペーシングは 4px 刻み。角丸は `radius-sm`=4px（ボタン・チップ）／`radius-md`=6px（パネル・ウィンドウ枠）。
* グループの外側余白 ≥ 内側余白。階層は「脇役を沈める」で作る（境界は影より明度差・罫線）。
* ツリー/タブは密度高め、プレビュー本文はゆとり。

## 5. 状態記号の体系

ファイルの状態は **色＋記号** で示し、**色だけに依存しない**。ツリー・タブで共通。

| 状態 | 記号 | 色 | 位置 | 備考 |
|---|---|---|---|---|
| 差分あり（変更） | `±` | accent（青） | 名前の右（ツリー）／名前の左（タブ） | 「未読＝ベースラインから変化＝差分あり」を一目で示す |
| 新規ファイル | `◆` | diff-add（緑） | 同上 | ベースラインなし＝全行追加。**差分行の `+` とは別記号**で混同を避ける |
| 削除済み | 取り消し線（記号なし） | 名前を `text-2` | — | 外部削除＋バッファ保持。取り消し線で削除を示す。**差分行の `−` は使わない** |
| 未保存 | `●` | accent（青） | タブの閉じる位置 | 位置（閉じるボタン）で「差分あり ±」と弁別 |
| 配下に差分あり | `±` | text-3（淡色） | フォルダ名の右 | 子孫の差分をフォルダへ伝播 |

* **重畳時の表示優先**：削除済み ＞ 未保存 ＞ 差分あり（要件5.3）。
* **差分ビュー内の行頭記号は別カテゴリ**：追加 `+`（緑）／削除 `−`（赤）。ファイル状態（±/◆/取り消し線）と記号体系を分け、混同を防ぐ。
* モノクロ・ハイコントラストでも、記号・位置・取り消し線で全状態を弁別できること。

### 5.1 エディタ内の可視化記号（特殊文字・装飾のみ）

文書を壊さず（正規化・変換せず）**見た目だけ**で気づかせる記号。ファイル状態記号（上表）とは別カテゴリ。要件5.1の「視認補助」を視覚化する。

| 対象 | 記号 | 色 | 意図 |
|---|---|---|---|
| 全角スペース（U+3000） | `□` | text-faint（さらに opacity 0.6 で控えめ） | 日本語字下げで頻出。半角と混ざると崩れる位置を知らせる。頻出ゆえ最も薄く |
| NBSP（U+00A0） | `·` | text-faint | コピペ由来の不可視文字。検索・整形を阻害するため可視化する。**ZWJ（U+200D）/ZWNJ（U+200C）は対象外**（絵文字結合 `👨‍👩‍👧‍👦` を `·` で分解しないため）。ZWSP（U+200B）/BOM（U+FEFF）等は CM6 既定で可視化される |
| 行末空白 | 背景の薄い警告ティント（記号なし） | alert を 16% 薄掛け | 意図せず残った末尾空白に気づく |
| 対応括弧／同一語ハイライト | ハイライト（記号なし） | 対応括弧＝弱い強調／同一語＝accent 16% 薄掛け | 同一語は Ctrl+F の検索ヒット（中立グレー）とは**色相で弁別** |

* いずれも**装飾のみ**でドキュメント文字は変えない（コピー・保存は元の文字のまま＝「データを失わない」）。
* 長行・巨大ファイル（要件2.2 の degrade）では、これら可視化・折りたたみ・シンタックスハイライトごと自動オフにして編集応答（200ms 予算）を守る。

### 5.2 エディタ編集コマンドのショートカット（キーボード）

要件 5.1/5.4 の編集補助に割り当てるキー。アプリ級ショートカット（要件11.2）とは別系統で、フォーカスがエディタにある間だけ効く（グローバルのショートカット解決には載せない）。

| 操作 | キー | 備考 |
|---|---|---|
| 太字トグル `**…**` | Ctrl+B | 選択を囲む／外す（往復で元へ） |
| 斜体トグル `_…_` | Ctrl+I | 旧 `selectParentSyntax` を上書き |
| 構文単位で選択を広げる | Ctrl+Shift+I | 旧 Ctrl+I の退避先 |
| チェックボックストグル | Ctrl+L | `- [ ]`↔`- [x]`（タスク記法行のみ） |
| 次の同一語を選択 | Ctrl+D | マルチカーソル追加 |
| 同一語を全選択 | Ctrl+Shift+L | — |
| 折りたたみ：畳む／開く | Ctrl+Shift+[ ／ Ctrl+Shift+] | Markdown 見出し単位 |
| 折りたたみ：全畳／全開 | Ctrl+Alt+[ ／ Ctrl+Alt+] | — |

* **折りたたみはキーボードのみ**。ミニマル維持のため fold gutter（折りたたみ余白の三角マーク）は表示しない（要件5.1）。

## 6. ファイルタイプアイコン（ツリー）

* **方針＝案A：モノトーン線アイコン**（lucide(MIT) 風）。`currentColor` でテーマ追従し、状態色（青/緑/赤）と干渉しない。
* 実装（**Tauri 刷新で改訂・2026-06-21**）：SVG を Web で表示（`<img src>` またはインライン SVG・CSS `currentColor` でテーマ追従）。高DPI は WebView2 が自動処理。`assets/icons/` に同梱（数十KB・30MB制約に無関係）。**lucide(MIT) ベース＋不足分のみ自作**（THIRD_PARTY_NOTICES にライセンス記載）。← 旧: wxBitmapBundle（wxWidgets 3.3）。
* **粒度＝カテゴリ集約＋代表専用**。未知拡張子は generic file アイコンへフォールバック。

| カテゴリ | 代表拡張子 | アイコン |
|---|---|---|
| フォルダ | — | folder |
| コード/マークアップ | ts/js/html/htm/xml | file-code（`<>`） |
| データ | json | braces（`{}`） |
| 設定 | yaml/toml/ini | sliders（設定）|
| スクリプト | sh/ps1/bat | terminal（`>_`） |
| 画像 | png/jpg/gif/webp/bmp/ico/svg | image |
| テキスト/文書 | md/markdown/txt/csv/log | file-text |
| 不明 | その他 | file（generic） |

* レイアウト：**左＝種別アイコン、右＝状態マーク（±/◆ 等）** で共存。アイコンは補助、ファイル名の拡張子が一次情報（色覚・スクリーンリーダ配慮）。
* マッピングの完全表は実装時に確定（要件5.1 のシンタックスハイライト対象種別と整合させる）。

## 7. レイアウト

構成（要件11.1）：メニューバー／左ペイン＝ファイルツリー（折りたたみ可）／タブバー／通知バー（必要時）／メイン領域／ステータス（右下固定）。ツールバーは持たない。

* **ファイルツリー**：絵文字は使わず、フォルダ開閉シェブロン（▾▸）＋種別アイコン＋名前＋状態マーク。選択は薄い背景（`bg-active`）のみ（アクセント線なし）。
* **ツリー折りたたみトグル**：展開時はヘッダ右端に収納ボタン「‹」。収納時は左端の細いレール（約30px）＋引き出しボタン「›」、メイン領域が拡幅（Obsidian 流）。
* **タブバー**：アクティブタブは面の明るさ（`bg-raised`）で示す（下線アクセントなし）。右端に **モード切替セグメント＋差分トグル＋「確認済みにする」** を配置（8章）。タブが画面幅を超えたらスクロール＋全タブ一覧ドロップダウン、隠れた差分ありタブはタブバー端のバッジで示す（要件5.3）。
* **メニューバー**：「表示」メニューに、エンコーディング指定/変更・改行コード・表示モード・折り返し・テーマを集約（9章）。
* **ウィンドウ枠（フレームレス・2026-06-24 改訂）**：OS 標準タイトルバーは持たず（`tauri.conf.json` `decorations:false`）、メニューバー帯がタイトルバーを兼ねる。空き帯（`data-tauri-drag-region`）でドラッグ移動し、帯右端に最小化/最大化(復元)/閉じるボタンを自前で置く（`@tauri-apps/api/window`）。リサイズは `resizable:true` のままで Tauri/WebView2 の枠ヒットテストが担う。これは**「全Web描画」補助原則**（design doc 1章）に沿った選択で、§1 原則4「ネイティブの見た目に馴染ませる」は Tauri 全Web刷新で「ネイティブ優先」が廃された経緯（本書冒頭・CLAUDE.md 設計原則）に従い、ウィンドウ枠についてはフレームレス＋自前導線を正とする。**要件14章「やらないこと」は増やさない**（機能追加ではなく既存ウィンドウ枠の見せ方の具体化）。

## 8. 表示モードと差分

**モードと差分を直交させる**（要件6.1 の改訂。14章参照）。

* **モード（排他）**：ソース／分割（エディタ＋プレビュー）／プレビュー。`.md`/`.html`/`.svg` で有効。初回既定はソース（6.1 改訂・2026-06-25。設定でプレビューに変更可）、以後はファイル種別ごとに記憶。
* **差分（独立トグル ON/OFF）**：モード切替セグメントの右に置く。ONは `accent-strong` 塗り（2章）。`Ctrl+Shift+D` でトグル。差分の中身は**常にテキスト（ソース）ベース**で、**描画面は常に WebView2 の差分HTML 1面**とする（design 6章「WebView2は1枚共有」と両立。Scintilla 側にインライン差分を新規実装しない＝軽量原則）。モードの違いは差分面に随伴するペインだけ：
  * **ソース＋差分ON** → 差分面のみ（実質的に差分ビュー。読み取り専用、編集は Ctrl+E でソースへ）。
  * **分割＋差分ON** → **プレビュー＋差分（左＝整形プレビュー／右＝差分面）**。実装は単一 WebView2 の左右 grid を流用し、プレビュー＋差分ON と同一表示に倒す（B11 仕様変更）。**理由：本ツールはレビュー用途で、分割＋差分で対比したいのは生ソースより整形プレビューと差分**（生ソースの編集は「ソースモード」または「分割＋差分OFF」で行う）。差分トグルを OFF へ戻すと「左＝エディタ／右＝整形プレビュー」の編集分割へ復帰する。
  * **プレビュー＋差分ON** → **1枚の WebView2 内HTMLに「左＝レンダリング、右＝テキスト差分」を横並び（grid）**で描く（共有1枚のまま左右同時表示を実現）。左右は独立スクロール（差分側はベースライン基準で現在内容と行対応しないため同期しない）。
* WebView2 の**占有世代（design 4章）は (タブ, モード, 差分ON/OFF) の組で数える**（差分トグルも WebView 内容を変えるため切替軸に含める）。
* **レンダリング済みプレビュー上には差分を描かない**（要件8.2／14章を維持）。
* 巨大ファイル（10MB超）・WebView2 不在・ベースライン未取得では差分トグルを無効化（自動オフ＋理由表示。通知バーから再有効化）。

## 9. ステータス表示（右下固定）

* 帯（フル幅バー）を廃止し、**メイン領域の右下隅に貼り付く灰背景**（`bg-sunken`）で表示。右辺・下辺は画面端まで、**左上だけ角丸**（Obsidian 流）。テキストは `text-2`（可読性確保。淡さは背景と小フォントで作る）。**表示専用**（対話はメニューへ。実装で `pointer-events:none` を意図的に維持）。
* **内容**：ワークスペースの**差分あり（未読）ファイル数**（レビュー進捗ゲージ・表示のみ）＋全体「行数・文字数」＋カーソル位置「行・文字目」。全体＝`text-2`、カーソル位置＝accent（青）で**色分け**。プレビュー/差分など編集カーソルがないモードでは全体のみ。巨大ファイルは「行数・サイズ」。文字選択中はカーソル位置の代わりに「選択 N 文字」。
* 文字を選択中はカーソル位置の代わりに「選択 N 文字」を表示（実装時）。
* **エンコーディング・改行コード・表示モードはステータスから外し、「表示」メニューへ移動**（Reopen with Encoding／Change Encoding 導線を維持。要件5.2・design 10章 G1）。

## 10. 通知バー

* 非モーダル・複数キュー（要件11.1）。**原則モノトーン**（`bg-sunken`＋罫線、ボタンは枠線）。
* **衝突（データ損失リスク）だけ**、左に細い警告バー（`alert`）＋見出しを alert 色で強調。外部リソース・JS検知・巨大ファイル・設定エラー等はモノトーン（種別は文言で示す）。
* 同時表示は最大3本＋「他N件」集約、種類の優先順位・集約・タブ固有/グローバルの扱いは要件11.1・design 10章に従う。

## 11. 差分ビュー

* インライン（unified）。行頭 `+`（緑）/`−`（赤）＋行背景（淡い緑/赤）。**色＋記号＋行内下線**で冗長化（色非依存）。
* 行内強調は下線＋太字。空白区切りが取れない文字列（日本語等）は文字単位で強調（要件8.2）。
* 「次の変更へ／前の変更へ」ナビ（F8/Shift+F8）。読み取り専用（編集は Ctrl+E でソースへ）。
* CSS は `assets/` の差分テンプレートへ流用する本命。

## 12. テーマ

* ライト／ダーク／システム追従（既定：システム追従。要件11.3）。`wxSysColourChangedEvent` で再起動なしに再適用（要件11.4）。
* エディタ・ツリー・Markdownプレビュー・差分にテーマを適用。**HTMLプレビュー（ユーザー文書）には適用しない**（文書自身のスタイルを尊重）。
* テーマの現在値（システム追従の解決結果）は `state.json` に保持（settings.toml は読み取り専用。design 7章）。

## 13. アクセシビリティ

* 全状態を色＋記号で弁別（±/◆/取り消し線/+/−）。低コントラストを避け、near-black/near-white を使う。
* ペイン間フォーカス循環（F6/Shift+F6・**フロントの自前フォーカスマネージャ**）、ハイコントラスト/テキストスケーリング追従（**`forced-colors: active`/`prefers-contrast` で独自トークン降格・rem/em 基準**）、主要UIにアクセシブルネーム（**ARIA**: `role="tree"/"tab"/"status"`・`aria-label`/`aria-live`）。要件11.5・design 10章・**design doc 17章（ARIA 全Web再構築）**。← 旧: UIA/MSAA（wx ネイティブ自動供給）。
* キーボードのみで「開く→プレビュー→差分→確認済み」が完結すること。

## 14. 要件・設計への反映（本書で確定し、本体を改訂する点）

本書の決定に伴い、`requirements.md`／`design.md` の以下を改訂する（整合を取る）。

1. **差分のモデル**（要件6.1／8.2／11.2、design 6章・`state.json.modeByType`）：
   「4モード（プレビュー/分割/ソース/差分）」→「**3モード（ソース/分割/プレビュー）＋差分トグル（ON/OFF・直交）**」。差分の中身はテキストベースのまま。プレビュー＋差分ON＝分割右にテキスト差分。**レンダリング済みプレビュー上の差分は引き続き「やらないこと」（要件14章）を維持**。`state.json` は表示モードと差分ON/OFFを別に保持。
2. **ステータスバー**（要件11.1）：フル幅バー→右下固定。内容を「**差分あり(未読)ファイル数**・行数・文字数・カーソル位置」に変更（未読件数の集計はステータス右下に残す＝レビュー進捗の一次情報）。エンコーディング・改行コード・表示モードは「表示」メニューへ移動し、**要件5.2（エンコーディング/改行の表示・再指定導線）と design 10章 G1 もメニュー基準へ追従改訂する**。11.1 のレイアウト構成行（メイン領域に差分を並記・「ステータスバー」表記）も新モデルへ更新。5.4 の検索モード依存も差分トグル語彙へ更新。
3. **未読＝差分あり**（要件4.2／5.3／8章）：UI 表現を「未読バッジ」から「差分ありマーク（±）」へ。新規＝◆、削除済み＝取り消し線で、差分行（+/−）と記号を分離。意味（外部変更＝未読＝差分あり）は不変。
4. **ツリー**（要件4.1）：ファイルタイプアイコン（モノトーン線・案A）を追加。折りたたみは収納/引き出しトグルで提供。
5. **色の使用ルール**（要件11.1/11.3）：モノトーン基調＋「色は差分・強調・衝突警告のみ」を視覚仕様として明文化。**【2026-06-28 改訂・ユーザー判断】ただしエディタの構文ハイライトはこの規則の例外**とし、`--syntax-*` で md/HTML/コードに色数を解禁する（§1 原則2・§2 トークン表）。UIシェルのモノトーンは維持。

これらは「機能の追加」ではなく**既存要件の見せ方の具体化・モデル変更**であり、要件14章「やらないこと」を増やさない（レンダリング差分・サイドバイサイド等の除外は維持）。

## 15. ビュー別の5状態（UI Stack）

全ビューで Ideal / Empty / Loading / Partial / Error を設計する（要件の受け入れ基準と対応）。モック実体は `docs/ui-mock.html` D節。

| 状態 | 代表シーン | 設計 |
|---|---|---|
| Ideal | フォルダを開きプレビュー/差分 | 7〜11章の通常表示 |
| Empty | フォルダ未オープン（初回）／検索0件／消化後 | 行き止まりにせず CTA 1つ＋最近使った項目。**3分岐で文言を変える**（要件10章） |
| Loading | ベースライン取得中・列挙中（1000中620 等） | percent-done＋件数。UI は非ブロック（design「固まらない」）。ツリーは逐次追加 |
| Partial（機能縮退/degraded） | 10MB超で差分・プレビュー自動オフ／ベースライン未取得のファイルを開いた | 黙って切らず理由＋手動再有効化を通知バーで明示（要件2.2）。**差分トグルは無効＋「差分基準を取得中」等の理由表示** |
| Error | WebView2 不在／アクセス権なし／衝突 | 機能を縮退してアプリは起動継続、次の一手を提示（要件2.3／12章） |

* 「機能縮退（degraded）」と「部分データ（partial）」を区別する。ベースライン未取得のファイルを開いた中間状態は差分トグルを無効化し理由を示す。
