# pika 手動受け入れチェックリスト（系統C）

> 本書は run-dev の UI 実装フェーズで自動化できない検証（系統C）を集約した手動チェックリスト。
> GUI 実機（wxWidgets ウィンドウ描画・Scintilla 実編集・WebView2 実描画・実 FS 監視・性能計測）が
> 必要で、ユニットテスト（系統A＝`ctest --preset x64-core-test`）・ビルド成立（系統B＝
> `cmake --build --preset x64-release`）では確認できない項目を扱う。dev/spec.md「検証戦略 系統C」と
> design.md 13章「結合テスト（半自動）」・14章3「性能は各スプリント末に計測」に対応する。
>
> 実施方法: Release ビルド（`build/x64-release/src/app/Release/pika.exe`）を基準機で起動し、各項目を
> 手で確認する。外部変更系は PowerShell の書き込みスクリプトで再現する（design.md 13章）。
> 各項目に対応する requirements / design / ui-design の節を併記し、判断のドリフトを防ぐ。
>
> 凡例: 結果欄に `[ ]` 未実施 / `[x]` 合格 / `[!]` 不合格（要修正）。計測値は実測を記入する。

---
